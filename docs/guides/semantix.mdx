description: Validate agent outputs against natural language intents

Marvin agents produce flexible, natural-language outputs — but flexibility can mean drift. semantix-ai lets you validate that an agent's response actually satisfies a given intent, using local NLI models with no API cost and ~15 ms latency per check.

## What is semantix-ai?

[semantix-ai](https://github.com/labrat-akhona/semantix-ai) is a semantic type system for AI outputs. Where schema validation checks structure ("is this a string?"), semantix validates meaning ("is this polite and professional?"). It runs entirely on-device via a small NLI model, so there are no external API calls and no data leaves your environment.

## Installation

```sh
pip install semantix-ai
```

No additional configuration is required. semantix downloads a small NLI model on first use and caches it locally.

## Inline Assertion

The quickest way to validate a Marvin agent's output is `assert_semantic`. It raises `AssertionError` (with score and reason) when the output fails the check:

```python
import marvin
from semantix.testing import assert_semantic

agent = marvin.Agent(
    name="SupportBot",
    instructions="You are a helpful customer support agent. Be polite and concise.",
)

output = agent.run("The customer is asking why their order is delayed.")

# Raises AssertionError with score and reason if it fails
assert_semantic(str(output), "must be polite, empathetic, and professional")
```

## Intent Classes and `@validate_intent`

For reusable constraints, define an `Intent` subclass — the docstring is the constraint — and decorate your agent call with `@validate_intent`:

```python
import marvin
from semantix import Intent, validate_intent

class Polite(Intent):
    """The text must be polite, empathetic, and professional."""

agent = marvin.Agent(
    name="SupportBot",
    instructions="You are a helpful customer support agent.",
)

@validate_intent
def support_reply(question: str) -> Polite:
    return agent.run(question)

# Returns the agent output; raises if the Polite intent is not satisfied
reply = support_reply("Why is my order late?")
```

## Self-Healing Retries

When `retries` is set, semantix automatically re-invokes the function on failure, injecting feedback into the optional `semantix_feedback` parameter so the agent can correct itself:

```python
from typing import Optional
from semantix import Intent, NLIJudge, validate_intent

class Polite(Intent):
    """The text must be polite, empathetic, and professional."""

@validate_intent(judge=NLIJudge(), retries=2)
def support_reply(question: str, semantix_feedback: Optional[str] = None) -> Polite:
    prompt = question
    if semantix_feedback:
        prompt += f"\n\nFeedback from previous attempt:\n{semantix_feedback}"
    return agent.run(prompt)
```

If all retries are exhausted the final validation error is raised.

## Composite Intents

Combine multiple intent classes with `&`. Every constraint must pass:

```python
from semantix import Intent

class Polite(Intent):
    """The text must be polite, empathetic, and professional."""

class Concise(Intent):
    """The response must be concise — no more than three sentences."""

PoliteAndConcise = Polite & Concise

@validate_intent
def support_reply(question: str) -> PoliteAndConcise:
    return agent.run(question)
```
